Overview

Designed and brought up MCU-based sensor test and data-collection PCBs with onboard memory, DMA-driven acquisition, and multiple communication interfaces to evaluate prototype MEMS inertial sensors at >12.8 kHz.

Constraints

  • High throughput without data loss
  • Multiple sensor variants + changing requirements
  • Flexible interfaces for lab + automation

Architecture

  • Onboard memory + SD logging
  • Interfaces: FDCAN, SPI, I²C, UART, USB3300
  • DMA pipelines + robust error handling

Results

  • Repeatable high-rate evaluation workflow
  • Validated multiple sensors concurrently
  • Enabled downstream noise characterization
