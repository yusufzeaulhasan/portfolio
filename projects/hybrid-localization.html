Hybrid Indoor Localization (IMU + UWB)

ESKF sensor fusion • asynchronous ranging • embedded implementation

Overview

Built a real-time indoor localization system that fuses high-rate IMU data with UWB ranging updates using an Error-State Kalman Filter (ESKF). The goal is robust tracking in GNSS-denied environments with practical timing constraints and intermittent measurements.

Constraints

  • IMU runs faster than UWB (mixed-rate updates)
  • UWB ranges arrive asynchronously per anchor
  • Need stable estimation under packet drops/outliers

Approach

  • Nominal state: position/velocity; attitude as quaternion
  • Error-state: \u03B4p, \u03B4v, \u03B4\u03B8 with ESKF updates
  • Per-measurement update: apply UWB correction as each range arrives

System

  • 4 anchors + mobile tag (UWB TWR ranging)
  • Embedded fusion loop with deterministic scheduling
  • Logging + offline evaluation in Python/MATLAB

Implementation Details (what to show)

  • Filter loop: IMU predict at fixed rate; UWB update on arrival (one update per anchor range)
  • Outlier handling: gating with innovation \u03C7\u00B2 threshold (or simple residual limit)
  • Time alignment: timestamp each measurement; propagate to measurement time before update
  • Outputs: position, velocity, attitude; optional bias states if you add them later
